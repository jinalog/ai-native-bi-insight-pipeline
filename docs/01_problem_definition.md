# 01. Problem Definition

## 1. 배경

광고·결제 기반 비즈니스 환경에서는
성과 하락이나 이상 징후가 발생했을 때
원인을 빠르게 파악하고 대응하는 것이 매우 중요하다.

그러나 실제 현업에서는 다음과 같은 문제가 반복된다.

- 광고 및 마케팅 운영자는 SQL을 직접 작성하기 어렵다
- 대시보드는 많지만 “왜”에 대한 해석은 여전히 사람의 몫이다
- 성과 하락이 발생해도 원인 분석에 시간이 소요된다
- 데이터 분석 요청이 데이터 팀에 집중되어 병목이 발생한다

이로 인해 의사결정 속도가 느려지고,
문제 대응이 사후적으로 이루어지는 경우가 많다.

---

## 2. 주요 사용자 정의

본 프로젝트의 주요 사용자는 다음과 같다.

- 광고 및 마케팅 성과를 관리하는 **비기술 직군 운영자**
- SQL, 데이터 모델, 테이블 구조에 대한 이해가 없음
- 다음과 같은 질문을 일상적으로 던짐

  - “어제 매출이 왜 떨어졌어?”
  - “CTR은 괜찮은데 전환이 안 나오는 이유가 뭐야?”
  - “비용은 늘었는데 ROAS가 떨어진 이유가 뭐야?”

사용자는 데이터를 직접 조회하거나
복잡한 리포트를 해석하는 것이 아니라,
**자연어 질문만으로 즉시 인사이트를 얻기를 기대한다.**

---

## 3. 해결하고자 하는 문제

이 프로젝트는 다음 문제를 해결하는 것을 목표로 한다.

1. 비기술 직군도 자연어 질문만으로
   광고·결제 성과를 분석할 수 있어야 한다
2. 성과 하락 및 이상 징후를
   단순 수치가 아닌 “원인 중심”으로 설명해야 한다
3. 데이터 조회 → 분석 → 해석 과정을
   하나의 자동화된 흐름으로 통합해야 한다

---

## 4. 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

- 광고 및 결제 로그를 기반으로 한
  **AI Native BI 파이프라인**을 구축한다
- LLM이 직접 질의 가능한 구조의
  **AI 친화적 데이터 마트**를 설계한다
- Text2SQL을 통해 자연어 질의를 SQL로 변환한다
- 분석 결과를 LLM 기반 요약으로 제공하여
  운영자가 즉시 이해할 수 있는 인사이트를 생성한다

---

## 5. 기대 효과

본 파이프라인을 통해 다음 효과를 기대한다.

- 운영자는 SQL 없이도 즉시 원인 분석 가능
- 반복적인 분석 요청 감소로 데이터 팀 부담 완화
- 이상 징후에 대한 선제적 대응 가능
- BI 대시보드가 “지표 조회 도구”가 아닌
  **의사결정 지원 도구**로 진화

---

## 6. 범위 및 제약

- 실제 서비스 데이터는 사용하지 않고 합성 데이터를 사용한다
- 모델 성능 최적화보다는
  **구조 설계와 흐름의 명확성**에 초점을 둔다
- 본 프로젝트는 프로덕션 구현이 아닌
  **설계 및 개념 검증 목적의 포트폴리오**이다
